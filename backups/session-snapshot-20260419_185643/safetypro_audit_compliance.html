RCA Technique
5-Why: Best for simple–moderate incidents. Ask "Why?" iteratively until the root cause is found.
Root Cause Identified
Ishikawa / Fishbone: Categorise causes under 6 M's. Effect (head of fish) = the incident.
Incident / Effect
ICAM (Incident Cause Analysis Method): Systematic analysis used globally for serious incidents. 4-category framework aligned with ISO 45001 Cl.10.2.
Fault Tree Analysis: Top-down deductive analysis. Start with the top event and work down to basic causes using AND/OR gates.
Bow-Tie: Shows threats \u2192 critical event \u2192 consequences with barriers on both sides. Identifies which barriers failed.
